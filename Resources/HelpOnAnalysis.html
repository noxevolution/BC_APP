Analysis Tools

The analytics tool allows you to study recorded timeseries as well as predicted their values.

Use drag and drop to pull timeseries from the selection tree to either of the timeseries comboboxes. You can also drop timeseries into the rule-based text box.

Several filters may be applied to the selected timeseries:

original

This shows the original timeseries without modification.

average

This shows a running average of the timeseries' value. You can specify the integration period to be used.

trend

This fits a trend line to the timeseries. You can specify the order of fit. An order of 1 is a straight line.

elasticity

This calculates elasticity, where elasticity is defined as:

(change in quantity * average price) / (change in price * average quantity)

You can calculate this over any number of samples.

rule-based

Here you can specify the transformation you wish to apply. There are some special notations you can use here:

  • i is the sample number. This runs from zero to the number of samples -1

  • original is the original timeseries

  • samples is the number of timesteps in the timeseries, including future timesteps

  • seriesCount is the number of timeseries in the database

  • To apply a lookup value use [1, 2, 3, 4, 5, 6, 7, 8][i]

  • Use if: e.g. 0 if i < 10 else 1

  • Use max () and min ()

  • Use fft (series, upperFrequency, lowerFrequency) to filter out frequency components of the timeseries, where f is the harmonic and the base period is the total length of the timeseries. The lowerFrequency argument is optional and defaults to zero. For example, fft({!}, 5)[i] returns the referenced timeseries with frequency components greater than or equal to 5 * base period removed. fft({!}, 5, 5)[i] returns just the fifth harmonic component. You can use this function to ascertain the periodicity components within a timeseries. It's hard to explain succinctly, best try it.

  • The evaluator differentiates between integer and real arithmetic

  • Reference timeseries by putting their full names inside curly brackets. If you drag a timeseries to the text box you will see the correct syntax. Preceding the name with an asterisk yields the series value at any timepoint. If you omit the leading asterisk you'll have to add [i] at the end of the reference timeseries. This latter form gives you the opportunity to alter the index value, e.g. replace i with i+1.

    e.g. {*My timeseries} or {MyTimeseries} [i-1]

  • {!} refers to the timeseries selected in the main combobox and {!*} refers to its elements as described above. {!}[i] is equivalent to {!*}.

You can access any of the following modules from within the rule-based text box: math, random.
